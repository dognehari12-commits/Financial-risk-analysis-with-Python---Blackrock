🚀 Project Overview:

This project analyzes customer transactional data to understand financial behavior, identify usage patterns, and detect potential risk indicators. Using Python, the project transforms raw financial data into meaningful insights that help support data-driven decision-making in financial institutions.

🎯 Objectives:

*Clean and preprocess financial transaction data

*Analyze credit vs debit transaction trends

*Identify high and low performing accounts

*Segment customers based on activity and balance

*Detect dormant and high-risk accounts

*Perform hypothesis testing on transaction behavior

🛠️ Tools & Technologies:

📂 Project Workflow:

*Data Cleaning & Formatting

*Descriptive Transaction Analysis

*Customer Profile Building

*Financial Risk Identification

*Visualization (EDA & Risk Charts)

*Hypothesis Testing

📈 Key Insights:

*Debit transactions dominate customer activity

*Majority of accounts fall under low activity

*High transaction frequency does not mean high balance

*A small percentage of accounts contribute to financial risk

*Balance volatility highlights unstable financial behavior

*Presence of dormant accounts indicates low engagement

💡 Business Recommendations:

*Implement real-time risk monitoring systems

*Re-engage dormant customers with targeted campaigns

*Offer investment products to high-balance customers

*Monitor high-frequency low-balance accounts

*Strengthen anomaly detection for fraud prevention
